# 产品路线图

> 摘要：本文件记录所有里程碑和迭代的规划状态。PM 在需求沟通阶段维护此文件。
> 完整产品需求参见 [PRD.md](PRD.md)。

## 版本策略

采用 **里程碑 + 小迭代** 模式：`vX.Y.Z`
- `X.Y` = 里程碑编号（对应产品模块主题）
- `Z` = 迭代序号（每次迭代覆盖 1-3 个功能点）
- 每次迭代走完整 7 阶段流程（需求→设计→后端→前端→集成→评审→测试）
- PM 在需求阶段从 PRD 中选取功能点组成本次迭代
