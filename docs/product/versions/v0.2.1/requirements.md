> 里程碑: v0.2.x（账号管理）

## 摘要

- **版本目标**: 平台自动定期同步所有抖音账号的基础信息和视频数据，员工也可在账号详情页手动触发同步
- **功能数量**: 1 个（F-001-2 账号信息同步）
- **优先级分布**: P0(1)
- **前置依赖**: v0.2.0（账号添加 + 列表展示）已完成；`DouyinVideo` 表 schema 已在 v0.2.0 中创建但尚无数据
- **后续迭代**: v0.2.2（F-002-1 收藏同步 + F-002-2 对标账号管理）

---

## 用户故事

**作为员工**，我希望平台能自动更新我的账号粉丝数、作品数等基础信息，  
**以便**我无需手动刷新也能看到最新的账号状态。

**作为员工**，我希望我发布的新视频能自动出现在视频列表中，  
**以便**我可以在平台上追踪每条视频的数据表现。

**作为员工**，我希望在账号详情页能手动触发同步，  
**以便**在需要最新数据时不必等待下次定时任务。

---

## 面向角色与访问权限

| 角色 | 权限说明 |
|------|----------|
| **员工** | 可在自己账号的详情页手动触发同步；查看同步状态和最后同步时间 |
| **分公司负责人** | 只读查看同步状态；不可触发同步 |
| **超级管理员** | 只读查看同步状态；不可触发同步 |

> 定时任务为系统级行为，不区分角色，对所有 `DouyinAccount` 全量扫描。

## 功能清单

### F-001-2: 账号信息同步 (P0)

本功能由两个独立的定时任务组成，外加一个手动触发入口，共三个子功能点。

---

#### F-001-2a: 账号基础信息同步定时器

**描述**: 系统后台以固定间隔全局扫描所有 `DouyinAccount`，调用爬虫更新账号基础信息字段。

**触发方式**: 自动定时（全局，按配置间隔运行）

**同步字段**（`DouyinAccount`）:

| 字段 | 说明 |
|------|------|
| `nickname` | 账号昵称（可能被用户修改） |
| `avatar` | 头像 URL |
| `followersCount` | 粉丝数 |
| `videosCount` | 作品数 |
| `bio` | 简介 |

> [PM-ASSUMPTION] 以上字段为账号主页可见的基础公开信息，与 F-001-1 添加时爬取的字段一致，通过再次调用同一爬虫接口刷新。

**执行逻辑**:
1. 定时器按配置间隔启动
2. 从数据库获取所有 `DouyinAccount`（不区分类型和归属员工）
3. 逐条调用爬虫接口获取最新账号信息
4. 将新数据写回对应记录，同时更新该账号记录上的「最后同步时间」字段
5. 单条账号爬虫失败时，记录错误日志，**跳过该条、继续执行其他账号**（不中断整批）

**失败处理**:
- 单次同步失败：跳过，不中断批处理，记录日志。下次定时器运行时自动重试
- 无需即时重试（下次定时任务即为重试机会）
- 同步失败不回滚已有数据，保留上次成功同步的数据

**频率配置**:
- 同步间隔由系统配置项控制（独立于视频同步定时器）
- [PM-ASSUMPTION] 建议默认值：每 6 小时同步一次，配置后不强制要求用户修改

---

#### F-001-2b: 视频增量同步定时器

**描述**: 系统后台以固定间隔全局扫描所有 `DouyinAccount`，调用爬虫增量抓取新发布视频，并更新已有视频的数据指标。

**触发方式**: 自动定时（全局，按配置间隔运行，**独立于账号信息定时器**）

**同步内容 — 新视频（新增记录到 `DouyinVideo`）**:

| 字段 | 说明 |
|------|------|
| `videoId` | 视频平台唯一 ID（抖音侧唯一标识，用于去重） |
| `title` | 视频标题 |
| `coverUrl` | 封面图 URL |
| `publishedAt` | 发布时间 |
| `videoUrl` | 视频播放链接 |
| `playsCount` | 播放量 |
| `likesCount` | 点赞数 |
| `commentsCount` | 评论数 |
| `sharesCount` | 转发数 |

**同步内容 — 已有视频（更新数据指标）**:

| 字段 | 说明 |
|------|------|
| `playsCount` | 播放量 |
| `likesCount` | 点赞数 |
| `commentsCount` | 评论数 |
| `sharesCount` | 转发数 |

**增量策略**:
- 以 `videoId` 为唯一键判断是否已存在
- 存在：更新数据指标字段
- 不存在：新增记录
- [PM-ASSUMPTION] 每次抓取账号最新发布的视频列表（按发布时间倒序），当遇到已存在且发布时间较旧的视频时可停止继续翻页（典型增量策略），具体翻页深度由技术层面决定

**执行逻辑**:
1. 定时器按配置间隔启动
2. 从数据库获取所有 `DouyinAccount`
3. 逐账号调用爬虫接口获取最新视频列表
4. 按增量策略写入/更新 `DouyinVideo`
5. 单个账号爬虫失败时，跳过、继续执行其他账号，记录日志

**失败处理**:
- 与 F-001-2a 一致：跳过失败账号、保留已有数据、下次自动重试

**频率配置**:
- 同步间隔由系统配置项控制（独立于账号信息定时器）
- [PM-ASSUMPTION] 建议默认值：每 1 小时同步一次（视频数据时效性较高）

---

#### F-001-2c: 手动触发同步

**描述**: 员工在账号详情页可手动触发针对该单一账号的同步（账号信息 + 视频），无需等待定时任务。

**触发方式**: 用户主动操作

**交互流程**:
1. 员工进入「我的账号」→ 点击某个账号卡片 → 进入账号详情页
2. 详情页显示：账号基础信息 + 视频列表 + 「最后同步时间」 + 「立即同步」按钮
3. 点击「立即同步」→ 按钮变为 loading 状态，显示「同步中…」
4. 同步完成（成功）→ 按钮恢复，页面数据刷新，「最后同步时间」更新为当前时间，显示成功提示
5. 同步失败 → 按钮恢复，显示错误提示「同步失败，请稍后再试」，「最后同步时间」不更新

**同步范围**: 仅针对当前账号，同时执行账号信息同步（F-001-2a 逻辑）和视频增量同步（F-001-2b 逻辑）

**防抖**:
- 同步进行中（loading 状态）禁用「立即同步」按钮，防止重复触发
- [PM-ASSUMPTION] 不设置冷却时间限制（前端 loading 态已足够防止快速重复点击）

---

#### F-001-2d: 同步状态可见性

**描述**: 用户在账号详情页可感知账号的同步状态和最近同步时间。

**展示内容**:

| 状态 | 用户可见信息 |
|------|------------|
| 从未同步 | 最后同步时间处显示「尚未同步」 |
| 同步中 | 按钮 loading 态 + 「同步中…」文字（仅手动触发时可见，定时任务在后台执行不展示） |
| 同步成功 | 「最后同步时间: N 分钟前 / HH:MM」 |
| 上次同步失败 | [PM-ASSUMPTION] 仅显示「最后同步时间」（上次成功的时间），不向普通用户暴露失败状态；失败信息记录在系统日志中供运维排查 |

> [PM-ASSUMPTION] 定时任务后台执行时，不在前端推送进度通知，不产生用户可见的同步状态变化（用户下次访问详情页时会看到已更新的数据和时间戳）。

## 验收标准

### 定时任务

1. 账号信息同步定时器（F-001-2a）按配置间隔自动运行，执行后账号的粉丝数、昵称等基础信息更新为最新值
2. 视频同步定时器（F-001-2b）按配置间隔自动运行，执行后：
   - 账号新发布的视频出现在视频列表中（新增 `DouyinVideo` 记录）
   - 已有视频的播放量、点赞数等数据指标更新
3. 某一账号爬虫失败时，该账号被跳过，其他账号的同步继续正常执行（不中断整批）
4. 两个定时器的运行间隔可独立配置

### 手动同步

5. 账号详情页有「最后同步时间」和「立即同步」按钮
6. 从未同步的账号，「最后同步时间」处显示「尚未同步」
7. 点击「立即同步」后，按钮进入 loading 状态，同步完成后数据刷新，最后同步时间更新
8. 手动同步失败时，有明确错误提示，现有数据和最后同步时间不变
9. 同步进行中，「立即同步」按钮不可再次点击

### 数据完整性

10. 爬虫失败不影响已有数据（不覆盖为空，不删除已有记录）
11. 同一 `videoId` 不产生重复 `DouyinVideo` 记录（幂等写入）

## PM 注解汇总

以下为本文档中所有 `[PM-ASSUMPTION]` 标注的合理补充，供需求评审时确认：

| 编号 | 假设内容 | 影响 |
|------|----------|------|
| A-01 | 账号信息同步通过与 F-001-1 相同的爬虫接口再次调用实现 | 后端无需对接新爬虫接口 |
| A-02 | 账号信息同步默认频率：每 6 小时一次 | 可由技术层调整配置默认值 |
| A-03 | 视频同步默认频率：每 1 小时一次 | 可由技术层调整配置默认值 |
| A-04 | 视频增量策略：遇到已存在且发布时间较旧的视频时停止翻页 | 技术层决定翻页深度逻辑 |

| A-07 | 定时任务后台静默执行，不在前端实时推送进度 | 降低实现复杂度；用户通过刷新/手动同步获取最新数据 |
