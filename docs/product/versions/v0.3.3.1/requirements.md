> **版本定位说明**：v0.3.3 的 AI 仿写严格要求视频必须存在拆解批注（Annotation）才可发起，此限制对于"没有对标视频、直接基于观点+主题创作"的场景不适用。v0.3.3.1 为 Rewrite 模型追加直接创作模式，在无 workspace（无拆解视频）的情况下，用户通过纯观点 + 创作主题/指令直接驱动 AI 生成文案。

## 摘要

- **版本目标**：提供独立于对标视频拆解的"直接创作"入口，用户选择今日观点、填写创作主题/指令、选择目标账号和 AI 模型后，由 AI 生成完整短视频文案；支持多次生成产出多个版本，每个版本可人工编辑。
- **核心差异（与 v0.3.3 的区别）**：
  - 无需 AiWorkspace（不绑定对标视频）
  - 无 Annotation（无拆解批注框架）
  - Prompt 以"创作主题/指令"替代"拆解框架"
  - UI 为单栏或两栏布局，不再是三栏
- **任务/版本口径**：
  - 每次开启新的 AI 直接创作，对应一个新的 `Rewrite` 创作任务
  - 同一个创作任务内，可多次点击「创作」生成多个 `RewriteVersion` 版本
  - 用户明确点击「新建任务」后，清空当前任务上下文，下一次生成创建新的 `Rewrite`
- **前置依赖**：
  - v0.3.3 的 Rewrite + RewriteVersion 数据模型已存在
  - v0.3.2 的观点库（Fragment 表）已存在
  - 系统 AI 配置中已配置至少一个 AiModelConfig
  - 员工已添加至少一个「我的账号」（DouyinAccount）
- **本版交付**：
  - Prisma schema 变更：`Rewrite.workspaceId` 改为可为空（nullable）
  - 新增 API：`POST /api/rewrites/direct/generate`、`GET /api/rewrites/direct/[rewriteId]`、`PATCH /api/rewrites/direct/[rewriteId]/versions/[versionId]`
    - `POST /generate` 首次生成不传 `rewriteId`，创建新的直接创作任务
    - `POST /generate` 继续生成传入 `rewriteId`，在同一任务下追加新版本
  - RewriteWorker 支持无 workspace 的"直接创作"分支（Prompt 不含拆解框架与原文）
  - 新增"直接创作"页面：独立页面（`/rewrites/new`），两栏布局（左侧输入区 + 右侧结果区）
  - 仪表盘 / 侧边栏快捷入口

## 边界结论

### 范围内

- Prisma schema：`Rewrite.workspaceId` 改为 `String?`（nullable），保留 `@unique`；MySQL 允许多个 NULL，因此可支持多个 workspace-free 记录
- 新增 `Rewrite.mode` 字段（WORKSPACE / DIRECT），其中 DIRECT 表示直接创作任务
- 新增 `Rewrite.topic` 字段（String?），存储直接创作时的创作主题/指令
- 直接创作页面（`/rewrites/new`）包含：
  1. 「选择今日观点」多选弹框（复用 `ViewpointPicker`）
  2. 已选观点 Chip 展示（可逐条移除）
  3. 临时素材输入框（促进创作的金句/例子）
  4. **创作主题/指令**输入框（核心新增，替代拆解框架）
  5. 目标账号选择（下拉，仅 MY_ACCOUNT 类型）
  6. AI 模型选择（下拉，全量 AiModelConfig）
  7. 版本 Badge/下拉（版本 N，切换历史版本）
  8. 「创作」按钮
  9. 生成结果编辑区（editable textarea）
  10. 「新建任务」按钮（清空当前 `rewriteId` 与结果区，保留或清空输入由前端交互实现决定）
- RewriteWorker 新增直接创作 Prompt 分支：以"创作主题/指令 + 观点素材 + 目标账号风格"构建 Prompt，不包含拆解框架和原文
- localStorage 持久化（按页面级存储：当前 `rewriteId`、选择的观点 IDs、临时素材、主题、模型 ID、账号 ID）
- 侧边栏或仪表盘提供"直接创作"快捷入口（图标 + 文字）
- 复用 `AiRewritePanel` 的版本切换、编辑自动保存逻辑

### 非目标（明确排除）

- 直接创作历史记录列表页（本版只维护当前任务上下文，不提供任务列表检索）
- 直接创作与对标视频关联（→ M-006 F-006-1）
- Prompt 模板管理（→ F-005-3）
- 「标记最终稿」按钮（→ 后续版本）
- 历史仿写经验注入 Prompt（→ M-006 后）
- 对标账号 BENCHMARK_ACCOUNT 类型的账号出现在目标账号下拉中

## 用户故事

### US-001：直接创作文案

**作为**员工，  
**我希望**在不选择对标视频的情况下，直接输入创作主题和观点素材，让 AI 生成一篇完整的短视频文案，  
**以便**我在有想法但没有对应对标视频时也能快速产出文案。

**验收标准：**
- AC-001：点击"直接创作"入口后，进入独立创作页面，无须选择对标视频
- AC-002：选择今日观点（可多选），Chip 区域正确展示已选观点并可逐条移除
- AC-003：输入"创作主题/指令"文本框，提示词不为空时「创作」按钮可用
- AC-004：选择目标账号和 AI 模型后，点击「创作」发起生成
- AC-005：生成期间输入区和按钮禁用，展示"生成中"状态
- AC-006：生成成功后，结果区展示 AI 生成的文案，可人工编辑

### US-002：多版本管理

**作为**员工，  
**我希望**能对同一直接创作任务多次生成，并在多个版本间切换，  
**以便**我对比不同版本并选择最合适的文案。

**验收标准：**
- AC-007：首次点击「创作」创建新的直接创作任务和版本 1；同一任务内再次点击「创作」生成新版本，版本编号自增
- AC-008：版本下拉/Badge 切换后，编辑区展示对应版本内容（editedContent ?? generatedContent）
- AC-009：已有编辑内容时再次点击「创作」弹出确认对话框
- AC-009a：点击「新建任务」后，当前结果区与版本状态清空；下一次点击「创作」创建新的直接创作任务

### US-003：自动保存编辑

**作为**员工，  
**我希望**我对文案的手动编辑能自动保存，  
**以便**不会因为误操作或页面刷新丢失编辑内容。

**验收标准：**
- AC-010：编辑区内容变更后，1 秒防抖自动保存为 `RewriteVersion.editedContent`
- AC-011：localStorage 保存当前 `rewriteId`、已选观点 IDs、临时素材、主题、模型 ID、账号 ID，刷新页面后恢复当前任务上下文

## 非功能要求

| 类型 | 要求 |
|------|------|
| 性能 | 生成请求异步（BullMQ 队列），前端 2s 轮询，预期生成时间 5-30s |
| 安全 | API 需 `auth()` 校验；targetAccount 必须属于调用者；仅员工及以上角色可调用 |
| 一致性 | 生成结果文案格式与 v0.3.3 工作台仿写结果一致，支持换行 |
