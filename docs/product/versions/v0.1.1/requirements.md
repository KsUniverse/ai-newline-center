# v0.1.1 需求文档 — 组织管理 + 用户管理

> 版本: v0.1.1
> 里程碑: v0.1.x (组织与权限)
> 需求来源: [PRD.md M-000](../../PRD.md) + 本次需求沟通补充

---

## 摘要

- **版本目标**: 实现管理员对组织结构和用户的完整管理能力
- **功能数量**: 2 个（F-000-2 组织管理、F-000-3 用户管理）
- **优先级分布**: P0(2)
- **前置依赖**: v0.1.0（登录功能）已完成

---

## 已确认业务规则（需求沟通补充）

| 规则 | 说明 |
|------|------|
| 组织层级 | 仅两层：集团（唯一）→ 分公司（多个），无更深层级 |
| 禁用组织 | 禁用分公司时，该分公司下所有用户账号**自动禁用** |
| 用户隔离 | 分公司负责人只能查看/管理**本公司**用户，不可跨公司 |
| 密码创建 | 管理员手动输入初始密码，无强制改密要求 |
| 用户归属 | 用户不可转移组织，归属关系创建后固定 |

## 功能清单

### F-000-2: 组织管理 (P0)

**描述**: 超级管理员管理分公司列表（查看、创建、编辑、禁用）

**访问权限**: 仅超级管理员可见此管理页面

**交互**:
1. 管理后台侧边栏 → 「组织管理」
2. 展示分公司列表（表格，含公司名称、状态、创建时间、负责人）
3. 点击「新建分公司」→ 右侧 Drawer 滑出 → 填写名称 → 提交
4. 点击某行 → 右侧 Drawer 展示详情/编辑（名称可修改）
5. 操作菜单 → 「禁用」→ AlertDialog 确认 → 确认后禁用（同时禁用其下所有用户）

**数据**:
- Organization（id、name、type、parentId、status、createdAt）
- 新增字段：`status`（ACTIVE / DISABLED）

**边界**:
- 集团组织不可创建、不可禁用、不可编辑（系统初始化时创建，只读展示）
- 禁用分公司时弹出确认提示，说明"该分公司下 N 个用户账号将同时被禁用"
- 已禁用的分公司可重新启用（用户账号不自动恢复，需手动单独启用）
- 分公司名称不可重复

**验收标准**:
1. 超级管理员可创建新分公司，创建后出现在列表中
2. 可编辑分公司名称
3. 禁用分公司后，该公司下所有用户登录被拒（账号状态变为 DISABLED）
4. 非超级管理员看不到组织管理入口

---

### F-000-3: 用户管理 (P0)

**描述**: 管理员/分公司负责人管理用户账号（查看、创建、编辑、禁用）

**访问权限**: 超级管理员（查看/管理全部用户）、分公司负责人（仅本公司用户）

**交互**:
1. 管理后台侧边栏 → 「用户管理」
2. 展示用户列表（表格，含姓名、账号、角色、所属分公司、状态）
3. 超级管理员可按分公司筛选；分公司负责人只看到本公司用户（无筛选器）
4. 点击「新建用户」→ 右侧 Drawer → 填写姓名、账号、密码、角色、所属组织 → 提交
5. 点击某行 → 右侧 Drawer 展示详情/编辑（姓名、角色可修改，账号不可修改）
6. 操作菜单 → 「禁用」→ AlertDialog 确认 → 立即生效

**数据**:
- User（id、account、passwordHash、name、role、organizationId、status、createdAt）
- 角色枚举：SUPER_ADMIN / BRANCH_MANAGER / EMPLOYEE

**边界**:
- 账号创建后不可修改
- 用户归属组织创建后不可修改（不支持转移）
- 分公司负责人只能创建 EMPLOYEE 角色用户（不能创建 SUPER_ADMIN / BRANCH_MANAGER）
- 超级管理员可创建所有角色
- 禁用用户后，该用户当前登录 Session 立即失效（下次请求被拦截）
- 不能禁用自己（当前登录账号）
- 账号全局唯一（不限组织）

**验收标准**:
1. 超级管理员可创建任意角色用户，并指定所属组织
2. 分公司负责人只能看到本公司用户，且只能创建 EMPLOYEE
3. 禁用用户后该用户无法登录
4. 用户列表按组织隔离，分公司负责人看不到其他公司用户

---

## 非功能要求

- 所有操作（创建/编辑/禁用）成功后显示 Toast 通知
- 列表支持分页（每页 20 条）
- 空列表显示 EmptyState 组件
- 操作中有 Loading 状态

---

## 页面与路由规划

| 路由 | 说明 | 权限 |
|------|------|------|
| `/admin/organizations` | 组织管理页 | SUPER_ADMIN |
| `/admin/users` | 用户管理页 | SUPER_ADMIN + BRANCH_MANAGER |

> 管理后台路由统一以 `/admin` 为前缀，通过 middleware 控制权限
