> **版本定位说明**：在 v0.3.1 建立 AI 工作台（AiWorkspace + AiDecompositionAnnotation）后，拆解结果散落在各对标账号的视频详情层中，员工无法跨账号、跨视频地纵览自己的拆解成果。v0.3.4 新增**拆解列表页**，以 AiWorkspace 为维度，将当前员工所有已进入工作台的视频聚合展示，支持筛选和排序，并提供直达工作台或发起仿写的快捷入口。

## 摘要

- **版本目标**：为员工提供一个全局拆解结果聚合视图，让其无须进入每个对标账号详情页即可快速浏览、筛选所有已拆解视频的批注成果，并快速跳转到对应 AI 工作台或直接发起仿写。
- **核心差异（与现有体验的区别）**：
  - 现有体验：批注结果只能在 `/benchmarks/[accountId]` 视频详情层内查看，无法跨账号汇总
  - 本版新增：独立的 `/decompositions` 列表页，展示当前员工全部 AiWorkspace 记录，支持多维筛选
- **前置依赖**：
  - v0.3.1 的 AiWorkspace、AiDecompositionAnnotation 数据模型已存在
  - v0.3.3 的 Rewrite 仿写流程（workspaceId 关联）已存在
  - BenchmarkVideo → BenchmarkAccount 关联链路完整
- **本版交付**：
  - 新增拆解列表页 `/decompositions`（员工维度，仅展示当前用户的 AiWorkspace）
  - 列表支持按对标账号筛选、按拆解状态（有批注/无批注）筛选、按时间排序
  - 每行展示：视频封面、视频标题、对标账号昵称、批注数量、拆解状态徽章、最后更新时间
  - 点击行：跳转至 `/ai-workspace/[videoId]`（进入该视频的 AI 工作台）
  - 仿写入口：当工作台已有批注（annotationCount > 0）时展示「发起仿写」快捷操作
  - 侧边栏新增「拆解列表」导航入口
  - Cursor 分页，每页 20 条

## 边界结论

### 范围内

- 员工维度的拆解列表页（`/decompositions`），每条记录对应一个 AiWorkspace
- 列表每行字段：
  - 视频封面图（BenchmarkVideo.coverUrl）
  - 视频标题（BenchmarkVideo.title）
  - 对标账号昵称与头像（BenchmarkAccount.nickname / avatar）
  - 批注数量（`_count.annotations`）
  - 拆解状态徽章：**有批注**（annotationCount > 0）/ **无批注**（annotationCount = 0）
  - 最后更新时间（AiWorkspace.updatedAt，相对时间展示）
- 筛选维度：
  - 对标账号（下拉多选，显示当前用户有 workspace 记录的所有 BenchmarkAccount）
  - 拆解状态（单选：全部 / 有批注 / 无批注）
- 默认排序：updatedAt 倒序（最近操作的在最前面）
- 点击行整体区域：打开 `/ai-workspace/[videoId]`，参数为 `BenchmarkVideo.id`
- 「发起仿写」快捷操作：每行末尾操作列，annotationCount > 0 时展示，点击后跳转至 `/ai-workspace/[videoId]?stage=rewrite`（直接打开工作台仿写阶段）
- 空状态：当前员工无任何工作台记录时，展示空状态引导（提示前往对标账号视频详情开始 AI 拆解）
- Cursor 分页，每页 20 条，页底 "加载更多" 或无限滚动触发

### 非目标（明确排除）

- 分公司负责人 / 超级管理员查看全组织拆解汇总（→ 后续管理功能迭代）
- 拆解列表内直接编辑或删除批注（→ 保持在 AI 工作台内完成）
- "已修正" / "AI 生成" 状态标记（PRD 原始定义；此处以 有批注/无批注 作为用户可感知状态，更简洁直观；已修正标记可在后续版本补充）
- 拆解结果作为仿写「框架」的选择弹框（PRD F-003-3 中提到，但需仿写流程重构支撑，→ M-005 完整实现时补充）
- 多选批量操作
- 拆解列表的搜索（按视频标题关键词）→ 可在后续迭代追加
- `/ai-workspace/[videoId]` 独立页面的实现（若不存在，技术架构师决定是新建路由还是复用对标账号详情层的路由跳转）

## 用户故事

### US-001：浏览所有已拆解视频的批注成果

**作为**员工，  
**我希望**在一个独立页面里看到我所有进入过 AI 工作台的视频及对应的批注数量、拆解状态，  
**以便**我能快速了解哪些视频已完成拆解、哪些还没有批注，而无需逐一进入每个对标账号详情页查找。

**验收标准：**

| # | 条件 |
|---|------|
| AC-001 | 列表只展示当前登录员工自己的 AiWorkspace 记录（userId = 当前用户，organizationId 隔离） |
| AC-002 | 每行展示：视频封面缩略图（宽高比 9:16 或与对标账号视频列表一致）、视频标题（最多两行截断）、对标账号昵称和头像（小头像+名称）、批注数量（数字）、拆解状态徽章（有批注 / 无批注）、最后更新时间（相对时间，如"3 小时前"） |
| AC-003 | 默认按 AiWorkspace.updatedAt 倒序排列；页面顶部提供"最近更新"排序下拉（本版仅此一种排序） |
| AC-004 | 使用 Cursor 分页，每页 20 条；页底展示"加载更多"按钮，无更多数据时隐藏 |
| AC-005 | 当员工无任何 AiWorkspace 记录时，展示空状态插图 + 说明文案（如"你还没有拆解过任何视频，前往对标账号视频列表开启 AI 工作台"）+ 跳转对标账号列表的链接按钮 |
| AC-006 | 侧边栏导航中"拆解列表"条目存在，点击跳转 `/decompositions` |

---

### US-002：按对标账号和状态筛选拆解结果

**作为**员工，  
**我希望**按对标账号或拆解状态（有批注/无批注）筛选拆解列表，  
**以便**我聚焦于特定博主的内容或快速找到还没完成拆解的视频。

**验收标准：**

| # | 条件 |
|---|------|
| AC-001 | 筛选栏位于列表顶部，包含两个筛选控件：「对标账号」多选下拉 + 「拆解状态」单选（全部 / 有批注 / 无批注） |
| AC-002 | 「对标账号」下拉仅列出当前用户有工作台记录的 BenchmarkAccount（按昵称展示，含头像），支持关键词搜索过滤下拉项 |
| AC-003 | 「拆解状态」筛选：有批注 = annotationCount > 0；无批注 = annotationCount = 0 |
| AC-004 | 筛选条件叠加生效（对标账号 AND 拆解状态同时生效）；筛选结果实时刷新列表（前端参数变化后重新请求） |
| AC-005 | 筛选条件已选时，筛选控件有视觉选中态；提供「清除筛选」快捷操作 |
| AC-006 | 无匹配结果时展示空状态（与 US-001 AC-005 的空状态文案区分，提示"没有符合条件的拆解记录"） |

---

### US-003：从拆解列表直接跳转到 AI 工作台

**作为**员工，  
**我希望**在拆解列表中点击任意一行，直接进入该视频的 AI 工作台，  
**以便**我能快速定位并继续拆解某个视频，而不需要经过对标账号 → 账号详情 → 视频列表的多级导航。

**验收标准：**

| # | 条件 |
|---|------|
| AC-001 | 点击行的任意区域（除操作列按钮外），跳转至 `/ai-workspace/[videoId]`（videoId 为 BenchmarkVideo.id） |
| AC-002 | 跳转后，AI 工作台恢复该员工此前的工作台状态（已有转录、已有批注均正常展示） |
| AC-003 | 鼠标悬停行时有高亮反馈，表明整行可点击 |

---

### US-004：从拆解结果一键发起仿写

**作为**员工，  
**我希望**在拆解列表中，对已有批注的视频直接点击「发起仿写」按钮，跳转到 AI 工作台的仿写阶段，  
**以便**我无需重新走拆解流程即可快速开始仿写。

**验收标准：**

| # | 条件 |
|---|------|
| AC-001 | 每行末尾操作列：annotationCount > 0 时展示「发起仿写」按钮；annotationCount = 0 时操作列为空或展示禁用态（不展示按钮，避免干扰） |
| AC-002 | 点击「发起仿写」跳转至 `/ai-workspace/[videoId]?stage=rewrite`，进入工作台仿写阶段（AI 工作台读取 `stage` 参数后自动切换到仿写步骤） |
| AC-003 | 跳转后仿写阶段的批注列表与工作台中的批注数据一致 |
| AC-004 | 若该工作台已有进行中的 Rewrite 任务，仿写阶段正常展示已有版本；不中断或重置已有仿写进度 |

## 非功能要求

| 类别 | 要求 |
|------|------|
| 数据隔离 | 列表查询必须同时过滤 `userId = 当前用户` + `organizationId = 当前用户组织`，二者缺一不可 |
| 性能 | 列表接口响应时间 < 800ms（P95），批注数量使用 `_count` 聚合，不返回原始批注列表 |
| 分页 | 使用 Cursor 分页（基于 AiWorkspace.updatedAt + id），不使用 offset 分页，避免大数据量下性能退化 |
| 权限 | 仅 EMPLOYEE、BRANCH_MANAGER、SUPER_ADMIN 角色可访问；BRANCH_MANAGER / SUPER_ADMIN 本版仍限于当前用户自身数据（不做角色提权差异） |
| 空封面容错 | BenchmarkVideo.coverUrl 可能为空，列表展示默认占位图（与对标账号视频列表一致） |
| 响应式 | 列表页适配桌面端宽度（≥ 1280px），与其他列表页风格一致（Linear 暗色主题） |
