## 摘要

- **版本目标**: 员工可添加自己管理的抖音账号，并查看账号列表及其视频列表
- **功能数量**: 2 个（F-001-1 添加抖音账号、F-001-3 账号与视频列表展示）
- **优先级分布**: P0(2)
- **前置依赖**: v0.1.1（组织管理 + 用户管理）已完成
- **后续迭代**: F-001-2（账号定时同步）将在 v0.2.1 实现

## 面向角色与访问权限

| 角色 | 权限说明 |
|------|----------|
| **员工** | 添加自己管理的抖音账号；查看**仅属于自己**的账号列表和视频列表 |
| **分公司负责人** | 只读查看**本分公司所有员工**的账号和视频列表；无法添加/删除账号 |
| **超级管理员** | 只读查看**全平台所有**账号和视频列表；无法代员工添加账号 |

> 数据隔离：员工只能操作和查看自己的数据（按 `userId` 过滤）；分公司负责人按 `organizationId` 过滤；超级管理员无过滤限制。

## 功能清单

### F-001-1: 添加抖音账号 (P0)

**描述**: 员工将自己管理的抖音账号的主页链接提交到平台，系统通过爬虫抓取账号基本信息后落库，后续视频数据由 v0.2.1 的定时同步任务负责抓取。

**访问权限**: 仅员工角色可操作（分公司负责人和超级管理员仅只读）

**定时器体系说明**（此处记录全貌以供架构设计参考，v0.2.0 不实现）:
> - **账号信息同步定时器**（v0.2.1 F-001-2）: 全局扫描所有 DouyinAccount，定期更新粉丝数、作品数等基础信息
> - **视频同步定时器**（v0.2.1 F-001-2）: 全局扫描所有 DouyinAccount，定期增量抓取新发布视频并写入 `DouyinVideo`
> - **收藏同步定时器**（v0.2.2 F-002-1）: 全局扫描员工的 `MY_ACCOUNT` 账号收藏列表 → 识别视频所属博主 → 自动创建对标账号（`type: BENCHMARK_ACCOUNT`）
> - **对标账号触发链**: 对标账号被创建后，自动纳入上述两个同步定时器的全局扫描范围（账号信息同步 + 视频同步），但**不递归触发收藏同步**（避免无限扩散）
> - 所有定时器采用**全局扫描模式**（按配置间隔扫描所有符合条件的账号），非按账号独立注册
> - 每类定时器运行间隔独立配置
>
> **v0.2.0 仅实现首次录入逻辑，不实现任何定时器。**

**交互流程**:
1. 「我的账号」页面 → 点击「添加账号」按钮
2. 右侧 Drawer 滑出 → 输入抖音主页链接（如 `https://www.douyin.com/user/xxxxx`）
3. 点击「获取账号信息」→ 系统调用爬虫（含 loading 状态）
4. 爬虫成功 → Drawer 中展示账号预览（头像、昵称、粉丝数、作品数、简介）
5. 用户确认预览信息 → 点击「添加」→ 系统仅保存**账号基本信息**（不抓取视频）→ Drawer 关闭 → 列表刷新

**数据实体**: `DouyinAccount`

| 字段 | 说明 |
|------|------|
| `profileUrl` | 抖音主页链接（唯一，用于去重） |
| `nickname` | 账号昵称 |
| `avatar` | 头像 URL |
| `followersCount` | 粉丝数 |
| `videosCount` | 作品数 |
| `bio` | 简介 |
| `userId` | 归属员工 ID |
| `organizationId` | 归属组织 ID（数据隔离使用） |
| `type` | `MY_ACCOUNT`（v0.2.0 仅此类型）|

> **v0.2.0 中初始添加不写入视频数据**，`DouyinVideo` 表在 v0.2.0 中创建 schema 但数据由 v0.2.1 定时器填充。
> 后续版本将新增 `BENCHMARK_ACCOUNT` 类型（v0.2.2 F-002-1 收藏同步自动创建的对标账号）。

**边界与异常**:
- **链接格式校验**: 提交前校验为合法的抖音主页 URL 格式，不合法时 inline 提示
- **重复检测**: 同一 `profileUrl` 全局唯一，不允许任何员工重复添加；若已存在，在 Drawer 中提示「该账号已被添加」
- **爬虫失败**: 显示错误提示（如「获取账号信息失败，请检查链接是否正确」），保留 Drawer 状态允许重试
- **多账号**: 一个员工可添加多个账号，无数量上限
- **初始视频为空**: 添加成功后视频列表为空，属于预期行为；待 v0.2.1 定时器首次运行后视频数据开始填充

**验收标准**:
1. 输入合法抖音主页链接后，能成功调用爬虫并在 Drawer 中展示账号预览信息
2. 点击「添加」后账号出现在「我的账号」列表中（视频列表为空，属预期行为）
3. 重复添加同一链接时给出明确提示且不创建重复记录
4. 爬虫失败时有明确错误提示，不创建任何记录

### F-001-3: 账号与视频列表展示 (P0)

**描述**: 展示员工（或在其权限范围内）的抖音账号卡片列表，以及点击进入后的视频列表页。

**访问权限**: 员工查看自己的；分公司负责人查看本公司所有员工的；超级管理员查看全部

**交互流程 — 账号列表**:
1. 左侧导航「我的账号」→ 进入账号列表页（`/accounts`）
2. 以卡片 Grid 布局展示账号，每张卡片显示：头像、昵称、粉丝数、作品数
3. 无账号时显示空状态（插图 + 引导文字「还没有添加账号，点击添加」+ 添加按钮）
4. 点击某个账号卡片 → 跳转账号详情页（`/accounts/[id]`）

**交互流程 — 视频列表（账号详情页）**:
1. 顶部展示账号信息区：头像、昵称、粉丝数、作品数、简介
2. 下方展示视频列表，默认按**发布时间倒序**排列
3. 视频列表以卡片或表格形式展示每条视频：封面缩略图、标题、发布时间、播放量、点赞数、评论数、转发数
4. 无视频时显示空状态（「暂无视频，账号信息正在同步中」）
5. 视频列表支持**分页**（每页 20 条）
6. 点击视频行/卡片 → 弹出视频详情 Dialog，显示完整信息（含视频原始链接）

**数据实体**: `DouyinVideo`

| 字段 | 说明 |
|------|------|
| `videoId` | 抖音视频原始 ID（唯一） |
| `accountId` | 归属账号 ID |
| `title` | 视频标题/描述 |
| `coverUrl` | 封面缩略图 URL |
| `videoUrl` | 视频原始链接 |
| `publishedAt` | 发布时间 |
| `playCount` | 播放量 |
| `likeCount` | 点赞数 |
| `commentCount` | 评论数 |
| `shareCount` | 转发数 |

**边界**:
- **员工**：只看自己添加的账号（`userId` 过滤）
- **分公司负责人**：查看**本分公司所有员工**的账号（`organizationId` 过滤），页面顶部标注「本公司账号」；分公司负责人无法添加账号
- **超级管理员**：查看全平台所有账号，无过滤
- **视频列表在 v0.2.0 为空**，属预期行为；待 v0.2.1 定时器运行后数据开始填充
- 视频封面图需做防盗链处理（通过后端代理或配置图片白名单，避免直接在前端 `<img>` 使用抖音 CDN 链接）

**验收标准**:
1. 账号列表以卡片形式展示，每张卡片包含头像、昵称、粉丝数、作品数
2. 无账号时展示引导性空状态
3. 进入账号详情页后，视频列表按发布时间倒序展示
4. 视频列表支持分页（每页 20 条）
5. 点击视频可查看详情弹窗
6. 无视频时展示空状态提示

## 自省报告

> 以下是需求整理过程中发现的**不明确点**，供后续确认或在架构设计阶段决策。

### ✅ 已确认问题

**Q1 — 初始添加时不抓取视频列表（已确认）**

首次录入时，仅调用爬虫抓取账号基本信息并落库。视频数据由 v0.2.1 的定时器体系（视频同步定时器）负责填充。v0.2.0 中视频列表为空属预期行为，`DouyinVideo` 表在本次迭代创建 schema 但暂无数据。

**Q2 — 分公司负责人查看全公司账号（已确认）**

分公司负责人可查看本分公司所有员工的账号，按 `organizationId` 过滤，前端页面顶部标注「本公司账号」。
