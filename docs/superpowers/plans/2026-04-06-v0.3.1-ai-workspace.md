# v0.3.1 AI Workspace Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v0.3.1 AI workspace, including system AI bindings, employee-owned transcript/decomposition/rewrite workspace data, real transcript generation from `shareUrl`, and the new analysis-first workspace UI.

**Architecture:** Keep the existing Route Handler -> Service -> Repository layering and reuse the current BullMQ/SSE transcription pipeline. Add a new employee-scoped AI workspace aggregate for transcript, segment, decomposition, and rewrite draft state, while replacing the benchmark detail entry point with an analysis-first workspace shell that preserves the current project design language, controls state through a single controller, and uses a shared-element style card-to-workspace transition.

**Tech Stack:** Next.js 15 App Router, TypeScript strict mode, Prisma 6, Vitest 3, Tailwind CSS 4, shadcn/ui, BullMQ 5, Zod 3
