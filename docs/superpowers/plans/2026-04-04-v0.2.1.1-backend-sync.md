# v0.2.1.1 Backend Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mock crawler logic with real crawler integration and deliver the complete `v0.2.1.1` backend foundation, including `secUserId`, snapshots, storage skeleton, and cross-account video listing.

**Architecture:** Keep the current Route Handler → Service → Repository layering and extend the existing Douyin account/video flows. Crawler field mapping stays inside `CrawlerService`, sync orchestration stays inside `SyncService`, and permission-aware pagination stays inside service/repository boundaries.

**Tech Stack:** Next.js 15 App Router, TypeScript strict mode, Prisma 6, PostgreSQL 16, Vitest 3, Zod 3, node-cron 3
